# 投资论点跟踪格式

## 文件模板

每个持仓或 watchlist 标的一个文件，默认存储在 `~/Documents/neoalpha/thesis-tracker/`。可用 `INVESTMENT_THESIS_DIR` 覆盖目录。

命名规则：
- 美股 thesis 保持 Longbridge 标准 symbol：`<SYMBOL>.US.md`，例如 `INTC.US.md`、`TLN.US.md`。
- 港股/A股 thesis 在 symbol 后追加股票名称：`<CODE>.<MARKET>-<股票名称>.md`，例如 `0700.HK-腾讯控股.md`、`300394.SZ-天孚通信.md`、`603986.SH-兆易创新.md`。

```markdown
# [TICKER] - [公司名]

## 基本信息
- **方向**: Long / Short
- **建立日期**: YYYY-MM-DD
- **成本价**: $XX.XX
- **当前价**: $XX.XX（更新日期）
- **目标价**: $XX.XX
- **止损价**: $XX.XX
- **当前 P&L**: +XX%

## 论点陈述
[1-2 句核心论点]

## 核心支柱
1. [支柱1] - 预期是什么
2. [支柱2] - 预期是什么
3. [支柱3] - 预期是什么

## 关键风险
1. [风险1] - 如果发生会怎样
2. [风险2] - 如果发生会怎样
3. [风险3] - 如果发生会怎样

## 评分卡
| 支柱 | 原始预期 | 当前状态 | 趋势 | 更新日期 |
|------|---------|---------|------|---------|
| | | | | |

## 更新日志
| 日期 | 数据点 | 论点影响 | 行动 | 信心度 |
|------|--------|---------|------|--------|
| | | | | |

## 催化剂日历
| 日期 | 事件 | 预期影响 | 备注 |
|------|------|---------|------|
| | | | |

## 研究笔记
[自由格式的额外观察、引用、链接]
```

## 审查频率
- **重大事件后**：立即更新
- **常规审查**：至少每季度一次
- **组合回顾时**：批量审查所有持仓

## 评分卡规则

- 每个 thesis 至少 3 个核心支柱。
- 每个支柱必须有“原始预期”和“当前状态”。
- 状态枚举：`Ahead` / `On track` / `Behind` / `Broken`。
- 趋势枚举：`改善` / `稳定` / `恶化`。
- 反证证据必须和具体支柱绑定，不能只写“有风险”。

## 反证证据规则

以下情况必须写入“反证证据”或“更新日志”：

| 类型 | 示例 |
|------|------|
| 财务 | 连续两季营收/利润低于 thesis 阈值 |
| 运营 | 客户流失、订单延期、产能爬坡失败 |
| 竞争 | 价格战、新进入者、份额流失 |
| 技术 | 替代路线被主流客户采用 |
| 治理 | 大额减持、审计/监管问题 |
| 市场 | SMAM Q5 且基本面也恶化 |

反证证据不是自动卖出信号，但必须触发 thesis 复核。

## 季度复核输出

```markdown
## Quarterly Thesis Review — [SYMBOL]

结论: Intact / Watch / Broken
质量评分: X → Y
估值: Base $X → $Y
主要强化证据:
1.
主要反证:
1.
操作影响:
- 维持 / 加入观察 / 降低仓位上限 / 移出 watchlist

以上仅供研究分析，不构成投资建议或交易指令。
```


## 写入规则

- 新建 thesis：只有用户明确要求“记录/建立/跟踪 thesis”时创建。
- 更新 thesis：重大事件后若用户要求更新，可直接写；若只是普通分析，先展示建议更新内容并确认。
- Merge 规则：保留原始论点，不覆盖历史；新增证据写入“更新日志”，改变结论时同步更新“当前状态/趋势”。
- 命名规则：美股使用 Longbridge 标准 symbol，格式 `<SYMBOL>.US.md`；港股/A股使用 `<CODE>.<MARKET>-<股票名称>.md`。脚本按文件名开头的 Longbridge symbol 识别标的。
- 禁止事项：不写入交易指令，不把目标价/止损价当作确定承诺；涉及交易动作必须标注“不构成投资建议或交易指令”。
