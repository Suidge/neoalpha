# Excel Conventions — Excel 建模通用规范

> 来源：anthropics/financial-services（Apache 2.0），适配个人投资者场景

## 环境适配

当前已具备 headless DCF `.xlsx` 生成与验证脚本，详见 [excel-toolchain.md](excel-toolchain.md)。普通研究默认仍先交付模型结构、公式设计、假设表和校验规则；用户明确要求文件时再生成 `.xlsx`。

生成 `.xlsx` 时按以下环境选择：

| 环境 | 写法 | 前提 |
|------|------|------|
| **Excel 内运行**（Office JS Add-in） | `range.formulas` 写公式，不单独重算；`range.format.*` 配色 | 用户已提供可操作的 Excel/Office JS 上下文 |
| **生成 .xlsx 文件**（Python/openpyxl） | 写公式字符串如 `ws["D20"] = "=D19*(1+$B$8)"` | 使用 `scripts/build_dcf_model.py`；用 `scripts/validate_dcf.py` 验证结构与公式 |

**禁止虚假承诺**：未用验证脚本通过前，不得宣称模型可交付；未用 Excel/LibreOffice 打开重算前，不得宣称 cached formula values 已刷新。

**合并单元格陷阱**：先对左上角单元格写值，再 merge + 格式化整块区域。先 merge 再写值会报 `InvalidArgument`。

## 公式优先于硬编码（不可协商）

- 所有预测、利润率、折现因子、PV、敏感性单元格**必须是 Excel 公式**
- 唯一允许硬编码的数字：(1) 历史实际值 (2) 假设驱动参数 (3) 当前市场数据
- 模型必须在用户修改任意假设后自动联动

## 分步验证原则

不要一口气建完再交付。每个关键阶段展示给用户确认：

1. 数据收集完成 → 确认原始输入
2. 预测完成 → 确认增长率与假设
3. 公式写完 → 确认联动正确
4. 敏感性表完成 → 确认中心格 = 基准值

## 敏感性表格规范

- 使用 **奇数行列**（标准 5×5），确保有真正的中心格
- 中心格 = 基准假设，输出必须等于模型实际结果（校验点）
- 行轴：WACC 变化；列轴：终端增长率 / EBITDA 利润率

## 颜色信号系统

### 字体颜色

| 颜色 | 含义 |
|------|------|
| 蓝色 `#0000FF` | 硬编码输入（手动填写的数字） |
| 黑色 `#000000` | 公式计算结果 |
| 绿色 `#008000` | 跨表引用链接 |

### 填充颜色

| 颜色 | 含义 |
|------|------|
| 深蓝 `#1F4E79` | 区块标题 |
| 浅蓝 `#D9E1F2` | 列标题 |
| 浅灰 `#F2F2F2` | 输入单元格 |
| 白色 | 公式单元格 |

**最小化配色**：只用蓝+灰+白+绿，不引入多色。如用户提供模板，优先跟随模板配色。

## 常见错误与修复

| 错误 | 原因 | 修复 |
|------|------|------|
| merge 后写值 | Office JS 报 pre-merge 维度 | 先写左上角，再 merge |
| Python 计算写数字 | 模型失去联动性 | 改写公式字符串 |
| 敏感性表偶数行列 | 无真正中心格 | 改用 5×5 或 7×7 |
| 多色配色 | 难以辨认输入/公式 | 只用蓝+灰+白+绿 |
