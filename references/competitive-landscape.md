# Competitive Landscape — 竞争格局与 Moat 分析

> 来源参考：Anthropic `financial-services` competitive-analysis，已改写为投资研究和 thesis 复核框架。

## 触发

"竞争格局"、"competitive landscape"、"竞品分析"、"对标公司"、"行业地位"、"moat"、"market map"、"谁是竞争对手"

## 一、适用场景

- 新主题或新行业研究，需要建立公司宇宙。
- 单一标的 thesis 需要验证行业地位和护城河。
- 可比公司选择存在争议，需要定义 peer set。
- 市场份额、商业模式、利润池正在变化。

## 二、先定义范围

| 问题 | 说明 |
|------|------|
| 分析对象 | 单一目标公司 / 多家公司横向对比 / 整个产业链 |
| 竞品集合 | 用户指定优先；未指定时先提出 5-15 个候选并说明原因 |
| 用途 | thesis、选股、行业概览、DCF/Comps、风险复核 |
| 深度 | 快速地图 / 标准研究 / 深度报告 |
| 投资问题 | 要回答的核心判断，例如"谁最受益于 AI capex？" |

## 三、行业关键指标

先找出行业真正运行的 3-5 个指标，再做比较。

| 行业 | 关键指标示例 |
|------|--------------|
| SaaS | ARR、NDR、CAC payback、Rule of 40、RPO |
| 半导体 | 制程/良率、产能、ASP、客户认证、毛利率 |
| 光通信 | 速率代际、出货量、客户结构、产能利用率、ASP |
| 电力/公用事业 | 装机、容量因子、PPA 价格、监管资产基数、利率敏感度 |
| 零售 | SSS、客流、客单价、库存周转、坪效 |
| 平台/市场 | GMV、take rate、供需两侧增长、留存、履约成本 |
| 医药 | 管线阶段、适应症空间、临床终点、专利期限、审批节点 |

## 四、分析流程

### Step 1: Market Context

| 维度 | 要求 |
|------|------|
| 市场规模 | TAM/SAM/SOM 分开，标注来源和日期 |
| 增速 | 历史 CAGR + 未来 CAGR，说明假设 |
| 驱动因素 | 需求、价格、政策、技术、资本开支 |
| 逆风 | 周期、监管、替代技术、价格压力 |

### Step 2: Value Chain / Profit Pool

画清价值链和利润池：

```text
上游 → 中游 → 下游 → 终端客户
成本/壁垒在哪一层？
利润率最高的一层是谁？
议价权正在向哪一层迁移？
```

输出表：

| 环节 | 代表公司 | 收入模式 | 毛利率区间 | 壁垒 | 当前趋势 |
|------|----------|----------|------------|------|----------|
| 上游 | | | | | |
| 中游 | | | | | |
| 下游 | | | | | |

### Step 3: Competitor Mapping

按最能解释竞争的维度分组：

| 分组方式 | 适用场景 |
|----------|----------|
| Business model | 平台、垂直、一体化、代工、渠道 |
| Segment | 企业/SMB/消费、低端/高端 |
| Posture | 直接竞争、相邻竞争、潜在进入者 |
| Geography | 中国、美国、欧洲、新兴市场 |
| Value-chain role | 设备、材料、设计、制造、服务 |

### Step 4: Positioning Visualization

不一定真的画图，但必须说明应该怎么画。

| 图形 | 使用条件 |
|------|----------|
| 2x2 matrix | 两个因素决定竞争位置，如增长/利润率、技术/成本 |
| Radar | 多维度综合评分，如产品、渠道、成本、品牌、研发 |
| Tier diagram | 自然分层明显，如龙头/第二梯队/长尾 |
| Value chain map | 垂直产业链 |
| Ecosystem map | 平台和网络效应行业 |

### Step 5: Company Deep Dives

每个关键公司至少包含：

| Metric | Value | Source |
|--------|-------|--------|
| Revenue | | |
| Revenue growth | | |
| EBITDA / EBIT margin | | |
| Market cap | | |
| Market share | | |
| Key product / segment | | |
| Core customers | | |

定性项：
- Business：一句话说明做什么。
- Strengths：2-3 条。
- Weaknesses：2-3 条。
- Strategy：当前优先事项。
- Recent moves：财报、并购、扩产、产品、价格。

### Step 6: Comparative Analysis

```markdown
| Dimension | Company A | Company B | Company C |
|-----------|-----------|-----------|-----------|
| Scale | ●●● $XB | ●●○ $XB | ●○○ $XB |
| Growth | ●●○ +X% | ●●● +X% | ●○○ +X% |
| Margin | ●●○ X% | ●○○ X% | ●●● X% |
| Moat | Strong | Moderate | Weak |
| Valuation | Premium | Discount | Fair |
```

评分必须显示实际数值，不能只给圆点。

### Step 7: Moat Assessment

| Moat | 检查问题 |
|------|----------|
| Network effects | 用户/供应商飞轮是否增强？是同边还是跨边网络效应？ |
| Switching costs | 技术集成、合同锁定、流程依赖、学习成本是否高？ |
| Scale economies | 单位成本是否随规模下降？最小有效规模多高？ |
| Intangible assets | 品牌、专利、许可证、数据、客户信任是否稀缺？ |
| Cost advantage | 资源、工艺、供应链、地理位置是否形成低成本？ |
| Regulation | 监管牌照是否保护存量玩家？ |

评级：Strong / Moderate / Weak。必须写一句证据。

### Step 8: Investment Synthesis

| 情景 | 概率 | 关键驱动 | 对目标公司的影响 |
|------|------|----------|------------------|
| Bull | | 份额提升、利润率扩张、估值重估 | |
| Base | | 当前趋势延续 | |
| Bear | | 价格战、技术替代、需求放缓 | |

必须回答：
- 谁在获得份额？
- 谁的 moat 最难被复制？
- 谁的估值已经反映了好消息？
- 哪些风险会破坏 thesis？

## 五、质量检查

- 所有公司指标来自同一财年或明确标注例外。
- 国际公司统一货币，并说明汇率日期。
- 每个关键数字有来源。
- 缺失数据用 `N/A` 或 `[E]`，不要留空。
- 估算值和公司披露值分开。
- 不把 TAM 当作可实现收入空间。
