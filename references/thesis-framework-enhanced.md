# Enhanced Thesis Framework — 基于 mingli30119/stock-analysis Step 0-8 体系

> **来源**: 整合自 mingli30119/stock-analysis (GitHub, MIT License) 的 Step 0-8 分析框架
> **定位**: financial 技能中 thesis 创建/更新的增强方法论，与现有 thesis 模板互补
> **适用**: 首次覆盖深度研究、重大事件重评、季度例行更新
> **最后更新**: 2026-05-16

---

## 一、框架定位

当前 thesis 模板（`skills/neoalpha/templates/thesis-template.md`）提供基础结构。本增强框架在现有模板之上加入：

1. **任务锁定与周期定位**（Step 0-1）— 在研究前明确分析边界
2. **产业链趋势三要素**（Step 2）— 供需 × 价格 × 海外验证
3. **五维质量评分**（Step 3）— 100 分制量化评估
4. **逻辑破坏条件**（Step 5）— 结构化止损信号
5. **三档目标价体系**（Step 6）— 短/中/长期分层

### 适用场景

| 场景 | 使用深度 | 说明 |
|------|---------|------|
| **首次覆盖** | 完整 Step 0-8 | 新建 thesis 文件 |
| **重大事件重评** | Step 0 + 1 + 4 + 5 + 6 | 催化剂变化时聚焦更新 |
| **季度更新** | Step 3 + 4 + 6 + 8 | 财报后刷新评分和目标价 |
| **动量扫描触发** | Step 6 估值-时机 | SMAM 信号触发时复核 |

## 二、Step 0-8 增强流程

### Step 0：任务锁定

在创建/更新 thesis 前，先明确四要素：

| 字段 | 可选值 | 写在 thesis 何处 |
|------|--------|-----------------|
| **标的** | 代码 + 名称 | 基本信息区 |
| **周期** | 短线(1-4周) / 中线(1-2季) / 长线(1年+) / 双周期 | 新增 `研究周期` 字段 |
| **研究状态** | 首次覆盖 / 跟踪更新 / 事件复盘 | 更新 `关注日期` 旁 |
| **风格预判** | 配置型 / 交易型 / 左侧博弈型 | 新增 `风格` 字段 |

### Step 1：宏观与周期定位

回答一个核心问题：**公司现在处在什么顺风/逆风？**

```
核心矛盾: [一句话，格式: X vs Y]
例如: "AI 算力需求爆发 vs 光刻胶国产验证爬坡"
```

**经济周期映射**（A 股适用）:

| 周期阶段 | 优势板块 | 对 thesis 标的影响 |
|---------|---------|-------------------|
| 复苏前期 | 有色/稀土/化工 | 原材料类标的受益 |
| 复苏中期 | 设备/制造 | 中游制造放量 |
| 复苏后期 | 消费/服务 | 下游消费跟随 |

### Step 2：产业链深度拆解 ⭐ 核心增强

#### 2a. 趋势三要素验证（每条业务线逐项）

| 业务线 | 产业阶段 | 供需格局 | 价格锚点 | 海外验证 | 结论 |
|--------|---------|---------|---------|---------|------|
| 主业A | 成长期/成熟期 | 供给收缩/需求扩张/供需平衡 | ASP 趋势 | 海外龙头同向/逆向 | 3/3 趋势确认 |
| 新业务B | 导入期/成长期 | ... | ... | ... | 2/3 待催化 |

**趋势三要素打分规则**:

| 要素 | 确认方法 | ✅ 条件 |
|------|---------|--------|
| 供需格局 | 供给是否收缩 + 需求是否新增 | 至少一侧偏紧 |
| 价格锚点 | 现货/期货/ASP 是否维持上行或高位 | 高于 12 月均线 |
| 海外验证 | 海外龙头财报、产能变动是否同向 | 同向确认 |

打分：3/3 = 趋势确认 / 2/3 = 可跟踪待催化 / ≤1/3 = 仅题材

#### 2b. 产业链图谱

```
产业链方向: [CPO光通信 / 半导体材料 / ...]
├── 上游: [原材料/设备/技术]
│   ├── 环节1 → 对公司影响
│   └── 环节2
├── 中游（★ 公司所在）
│   ├── 核心业务A（营收占比XX%，毛利率XX%）
│   ├── 成长业务B
│   └── 培育业务C
├── 下游: [客户/应用场景]
│   ├── 客户类型A（占营收XX%）
│   └── 客户类型B
└── 关键判断
    ├── 壁垒判断
    └── 利润迁移方向
```

### Step 3：质量评分（100 分制）⭐⭐ 核心增强

| 维度 | 权重 | 检查项 | 评分 |
|------|------|--------|------|
| **基本面质量** | 30 | 营收趋势、净利趋势、现金流、毛利率、负债率 | /30 |
| **产业匹配度** | 20 | 主营与主线相关度、产业地位、护城河 | /20 |
| **业绩弹性** | 20 | 量价敏感度、产能利用率、新业务放量空间 | /20 |
| **估值与位置** | 15 | PE/PB 分位、PEG、同业比较 | /15 |
| **治理与风险** | 15 | 管理层行动、减持、关联交易、诉讼风险 | /15 |
| **总分** | **100** | | **/100** |

**评级**: A(80-100) / B(60-79) / C(40-59) / D(<40)

**写入规则**: 首次覆盖给出完整五维评分；季度更新刷新基本面和估值维度；事件复盘只更新受影响维度。

### Step 4：业绩弹性测算

#### 4a. 弹性树（ASCII 格式）

```
公司: 鼎龙股份（300054）
核心资产: CMP抛光垫龙头 + 光刻胶平台

├── 业务A: CMP抛光垫（营收占比~30%）
│   ├── 产能: 5万片/月 → 预计扩至8万片/月
│   ├── 毛利率: ~45%
│   └── 弹性来源: 晶圆厂扩产→耗材跟随增长
│
├── 业务B: 光刻胶（营收占比~5%，快速爬坡）
│   ├── 产能: 300吨/年（潜江已投产）
│   ├── 毛利率: ~50%+（高端产品）
│   └── 弹性来源: 每多一款通过验证→新增收入曲线
│
└── 结论: 光刻胶是关键弹性点，每款验证=非线性增长
```

| 情景 | 触发条件 | 2027E 营收 | 2027E 净利 | 对应 PE(@现价) | 目标价 |
|------|---------|-----------|-----------|---------------|--------|
| 悲观 | | | | | |
| 基准 | | | | | |
| 乐观 | | | | | |

#### 5b. 逻辑破坏条件（止损信号，至少 3 条）

每条必须**可量化或可观测**，不含糊：

```
1. [量化] 核心产品 ASP 跌至 XX 以下
2. [事件] 大客户订单取消 >XX%
3. [财务] 连续两季营收增速 <XX%
4. [技术] 替代技术路线获主流客户采用
5. [治理] 创始人/核心高管减持 >XX%
```

**与 SMAM 联动**: 当 SMAM 发出 Q4/Q5 动量预警时，对照逻辑破坏条件——如果条件触发 + 动量恶化 → 执行清仓。如果条件未触发仅动量走弱 → 减仓观望。

### Step 6：估值-赔率与时机

#### 6a. 三档目标价（短/中/长期）

| 档位 | 时间框架 | 目标价 | 逻辑 | 向上空间 |
|------|---------|--------|------|---------|
| 短期 | 1-3 个月 | | 技术面/事件催化 | |
| 中期 | 6-12 个月 | | 业绩兑现/估值重估 | |
| 长期 | 12-24 个月 | | 产业趋势全面兑现 | |

#### 6b. 盈亏比量化

```
当前: ¥XX
向下空间: -XX%（至止损位）
向上空间: +XX%（至中期目标）
盈亏比: X.Xx

判断: ≥2.0x → 优质 / 1.5-2.0x → 合理 / <1.5x → 性价比不足
```

#### 6c. 与 SMAM 的协同

- SMAM CMS > +0.84 (Q1) → 动量窗口确认，可积极执行建仓/加仓
- SMAM CMS < -0.84 (Q5) → 动量预警，对照 Step 5 逻辑破坏条件
- VM_Score 综合正 + CMS 正 → 最佳入场时机

## 四、使用协议

### 首次覆盖（新建 thesis）
1. 运行完整 Step 0-8 流程
2. 数据源: Longbridge CLI 优先（报价/财务/新闻/K线/估值分位）
3. SMAM 扫描获取动量分位
4. 写入带增强字段的完整 thesis 文件
5. 质量评分必须基于当季最新财务数据

### 季度更新（维护 thesis）
1. 刷新 Step 3（质量评分）、Step 4（弹性/情景）、Step 6（目标价）
2. 对照 Step 5 逻辑破坏条件，检查是否触发
3. 运行 SMAM 扫描，更新动量追踪日志
4. 在 thesis 文件末尾追加 `## 更新日志` 条目

### 事件驱动（催化剂响应）
1. 聚焦受影响 Step：事件 → 对应 Step → 更新
2. 不重新走完整 8 步
3. 如果事件改变核心矛盾 → 必须更新 Step 1
